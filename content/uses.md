> A snapshot of the tools I reach for most often. This is not an exhaustive list and will drift over time

{{< lastmod >}}

## Writing and Notes

- **Obsidian** for notes, rough outlines and capturing ideas before they disappear.
- **Todoist** for turning "I should write that up" into something that might actually happen.

## Terminal and Development

- **macOS** as the daily driver.
- **iTerm2** and **zsh** for most command-line work.
- **Git** and **GitHub CLI** for day-to-day repo work.
- **Hugo** for this site, because writing in Markdown and shipping static files still feels correct.

## Infrastructure and Delivery

- **Terraform** for infrastructure changes.
- **GitHub Actions** for build and deploy automation.
- **AWS** for the actual hosting stack behind the site: S3, CloudFront and Route53.
- **Vercel** for pull request previews.

## AI and Experiments

- **Claude Code** and other agentic coding tools when they speed up the boring parts without making a mess.
- **MCP-based tooling** for focused workflows, especially around code review and repo automation.

## Hardware

- **A MacBook** for the main machine.
